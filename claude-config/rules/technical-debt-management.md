# Technical Debt Management — Measure, Prioritize, Pay Down

> Comentario en español: regla para cuantificar, priorizar y pagar deuda técnica de forma
> sistemática. Complementa `no-legacy-rule.md` (que cubre código zombie/legacy específicamente)
> y `cleanup-decision-tree.md` (que decide eliminar/consolidar/refactor/rewrite).
> Esta regla genera el REGISTRO de deuda + estrategia de pago.

## Categorías de deuda técnica

| Categoría | Ejemplos | Detección automática |
|---|---|---|
| **Code debt** | duplication, complexity, dead code, `_legacy_*`, comented-out blocks, magic numbers | `audit-module` skill + grep + AST |
| **Design debt** | god classes, circular deps, mixed concerns, missing layers | structural analysis + `cleanup-decision-tree.md` |
| **Test debt** | coverage <80%, flaky tests, no characterization tests on critical paths | coverage report + flakiness tracker |
| **Documentation debt** | stale CLAUDE.md, missing ADRs, outdated README, undocumented APIs | `claude-md-freshness.md` Stop hook |
| **Dependency debt** | libs >12 months behind, deprecated APIs in use, CVEs unfixed | `npm outdated` / `pip list --outdated` / `cargo outdated` + `npm audit` / `pip-audit` / `cargo audit` |
| **Infrastructure debt** | manual deploys, config drift, missing IaC, secrets in plaintext | infra scan + `compliance-pii.md` |
| **Knowledge debt** | bus factor 1, undocumented decisions, tribal knowledge | git blame + ADR coverage analysis |
| **Security debt** | hardcoded secrets, SQL injection, missing auth, weak crypto | `security-auditor` agent (OWASP Top 10) |
| **Performance debt** | N+1 queries, missing indexes, unbounded queries, no caching | `performance-at-scale.md` audit |
| **Compliance debt** | unclassified PII, missing audit log, GDPR/SOX gaps | `compliance-auditor` agent (ERP only) |

## Cuantificación (mandatory metrics)

Para cada repo, calcular y trackear:

```
📊 Tech Debt Score: <total>

CODE DEBT (auto):
  - Cyclomatic complexity > 10:        N functions
  - Files > 800 lines:                 N files
  - Functions > 50 lines:              N functions
  - Nesting depth > 4:                 N occurrences
  - Duplication ratio:                 X% (target <5%)
  - Dead code units:                   N functions/classes (zero call-sites)
  - `_legacy_*` files:                 N (regla no-legacy → eliminate)
  - Commented blocks >5 lines:         N
  - TODO/FIXME unresolved >90 days:    N

TEST DEBT:
  - Line coverage:                     X% (target ≥80%)
  - Branch coverage:                   X% (target ≥70%)
  - Flaky tests (failed >2x in last 30 runs): N
  - Critical path coverage:            X%

DEPENDENCY DEBT:
  - Direct deps outdated (>6 months):  N
  - Direct deps with CVEs:             N (CRITICAL/HIGH)
  - Deprecated APIs in use:            N occurrences

DOCUMENTATION DEBT:
  - Modules without CLAUDE.md (criteria met): N
  - CLAUDE.md stale (>90 days behind code): N
  - Broken refs in docs:               N
  - Missing ADRs for big decisions:    N

SECURITY DEBT:
  - Hardcoded secrets:                 N (CRITICAL)
  - SQL injection risks:               N
  - Missing auth on PII endpoints:     N
  - Weak crypto (MD5/SHA1 for pwd):    N

PERFORMANCE DEBT:
  - N+1 query patterns:                N
  - Unbounded list endpoints:          N (no pagination)
  - Missing indexes on FKs:            N
  - Cache without invalidation:        N
```

Each category gets a SEVERITY weight (CRITICAL=10, HIGH=5, MEDIUM=2, LOW=1) → total Tech Debt Score.

## Priorización (impact × effort matrix)

```
                LOW EFFORT          HIGH EFFORT
HIGH IMPACT  [Quick wins        ] [Strategic       ]
             [→ DO FIRST        ] [→ Plan + ADR    ]
LOW IMPACT   [Opportunistic     ] [Defer / skip    ]
             [→ When you touch  ] [→ Document, not act]
             [   that area      ] [                ]
```

Apply per item:
- **HIGH × LOW**: pay immediately (e.g., delete dead code, generate missing CLAUDE.md).
- **HIGH × HIGH**: schedule with ADR + tests + rollback plan (e.g., refactor a god class, ORM swap).
- **LOW × LOW**: pay when you're already in that file ("boy scout rule").
- **LOW × HIGH**: do NOT pay. Document and accept. Re-evaluate every 6 months.

## Pago de deuda — workflow

For each tech debt item, follow this sequence:

1. **Detect**: `audit-tech-debt` skill scans repo, updates registry.
2. **Classify**: assign category + severity + impact + effort.
3. **Prioritize**: place in 2x2 matrix above.
4. **Plan**: for CRITICAL/HIGH, generate a plan (test coverage first if needed).
5. **Pay**: apply `cleanup-decision-tree.md` action (eliminate / consolidate / refactor / rewrite).
6. **Verify**: tests pass, build OK, no regressions (`verification-first.md`).
7. **Document**: update CLAUDE.md, generate ADR if architectural decision.
8. **Close**: mark item resolved in registry with commit SHA.
9. **Audit drift**: weekly/monthly re-scan to ensure score is decreasing.

## Boy Scout Rule (always active)

> "Leave the code cleaner than you found it."

When you touch a file for ANY reason (feature, bug fix, refactor):
- Look for LOW × LOW debt items in that file.
- Pay them as part of the same commit (or split as related commit).
- Do NOT scope-creep beyond the file you touched.

This compounds — small wins per commit accumulate into massive cleanup over months without dedicated cleanup sprints.

## Integración con `no-legacy-rule.md`

`no-legacy-rule.md` cubre UNA subcategoría: **dead code / legacy / zombie / duplicate code in scope of current task**. La regla manda eliminarlo automático cuando es código tocado en la tarea.

Esta regla (`technical-debt-management.md`) es más amplia:
- Trackea TODA la deuda (code + test + docs + deps + security + perf + design + knowledge).
- Mantiene un REGISTRO vivo (no solo "elimino al pasar").
- Prioriza con matriz impact×effort.
- Decide cuándo pagar vs cuándo posponer (decision por costo de oportunidad).

Cuando se eliminan archivos legacy, se actualiza el registro automáticamente:
- TD-### items relacionados marcados como resolved con SHA.
- Score recalculado.

## Integración con `cleanup-decision-tree.md`

El árbol Q1-Q7 decide la ACCIÓN para cada item del registro:
- TD-### con call-sites = 0 → Q1 → ELIMINATE (auto if generated by IA, ask if preexisting).
- TD-### duplicado → Q2/Q3 → CONSOLIDATE.
- TD-### bad signature pero usado mucho → Q4 → REFACTOR INTERNAL.
- TD-### bad signature poco usado → Q5 → REWRITE + migrate.
- TD-### estructura mala → Q6 → REFACTOR BY LAYERS.
- TD-### sin tests → Q7 → ADD TESTS first.

## /audit-tech-debt slash command (auto-generated)

Available in templates. Usage:

```

```

Where `<scope>` is empty (whole repo), a module path, or a glob.

The IA:

2. Classifies + prioritizes each item.
3. Updates `docs/tech-debt.md` registry with diff.

## Cuando IA paga deuda automáticamente vs pide permiso

**Automático (en el scope tocado en la tarea actual)**:
- Eliminar dead code que YO genero o toco.
- Eliminar imports sin uso.
- Auto-format al editar.
- Generar CLAUDE.md faltantes en módulos que toco.
- Actualizar refs rotos.

**Requiere OK explícito (preexistente, no relacionado con tarea)**:
- Eliminar archivos `_legacy_*` preexistentes con call-sites.
- Refactor por capas de un módulo.
- Bump de major version de dependencia.
- Cambio de schema BD.
- Eliminación de funciones públicas (ROI cycle requerido).
- Cualquier item CRITICAL en critical path (auth/payments/billing).

## Anti-patterns

- ❌ "Tech debt sprint" mítico que nunca llega. Pagá continuo (boy scout rule).
- ❌ Trackear deuda en Jira/Linear sin link al código real. Usar `docs/tech-debt.md` con file:line.
- ❌ Pagar LOW × HIGH (effort grande, impact bajo). Costo de oportunidad alto.
- ❌ Pagar deuda sin tests previos (introducís regresiones invisibles).
- ❌ Pagar deuda + agregar feature en mismo commit. Siempre separar.
- ❌ Big-bang debt payment (1 PR refactor de 50 archivos). Incremental siempre.
- ❌ Ignorar deuda de seguridad / compliance porque "no es urgente". CRITICAL son siempre urgentes.
- ❌ No re-medir score después del pago. Sin medición no hay mejora demostrable.
